RECORDING OUTPUTS
-----------------
Running combined teleop logging produces two paired artifacts:

- logs/teleop_data_N.hdf5
  Structured datasets used by replay/analysis.

- logs/teleop_metadata_N.txt
  Human-readable metadata summary for the matching HDF5 file.

If --log-path is provided (for example demo.hdf5), the sidecar becomes demo.txt.

RUNTIME FILES
-------------
teleop_recorder.py
  - Live teleop + async logging process
  - Writes HDF5 plus metadata TXT sidecar

replay_hdf5.py
  - Replays arm/hand trajectory from HDF5
  - Optional video display when camera dataset exists

robot_pose_controller.py
  - Sends direct arm pose commands with safety filtering

verify_sync.py
  - Visual validation of camera and trajectory timing

realman_utils.py
  - Tracker-to-robot mapping and safety logic

track.py
  - Vive/OpenVR tracker module

QUICK COMMANDS
--------------
Record:
  python teleop_recorder.py

Replay:
  python replay_hdf5.py logs/teleop_data_0.hdf5

Dry-run inspect:
  python replay_hdf5.py logs/teleop_data_0.hdf5 --dry-run

Sync visualization:
  python verify_sync.py logs/teleop_data_0.hdf5

Single pose command:
  python robot_pose_controller.py --pose 0.3 0.0 0.2 0.0 0.0 0.0
